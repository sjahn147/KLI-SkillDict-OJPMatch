
### 처리 진행 상황
- CPU 속도 2.8GHz 기준 10,000 행/1시간 속도로 처리하고 있음. 동시 세션을 최대 8개까지 진행할 수 있어서 1시간에 8만 건씩 처리하고 있음.
- 12시간 경과했을 때 100만 건을 처리하였음. 일반 데스크톱 한 대로 전체 파일 500만 건을 처리하는 데 5일 정도가 소요되는 것으로 보임.
- 여러 대의 컴퓨터를 병행하여 작업하는 것으로 처리 속도를 크게 개선할 수 있음.
- Synology NAS에 자동으로 쌓이도록 세팅할 예정

### 처리 성능 개선
- CPU 성능 요구
  - pandas,re, rapidfuzz만 쓰기 때문에 처리 속도는 CPU 속도에 가장 큰 영향을 받음.
  - 예) 2.80GHz가 1.80GHz보다 두 배 더 빠르게 처리.
  - 동시 처리 가능한 세션 수는 CPU 용량에 의존.

- 메모리 요구
  - 메모리 부담을 줄이기 위해서는 200,000 행 짜리 파일을 그대로 얹어서 계산하면 안 되고, 최소 10,000개씩 잘라서 처리해야 여러 세션을 동시에 진행할 수 있음. 메모리 부담이 초과되면 세션이 종료됨.

- 배치 처리 / 멀티프로세싱
  - 멀티프로세싱을 위해 스킬을 배치 파일로 쪼개서 시도해보았지만 보통 사양의 개인 컴퓨터에서는 순차 처리가 더 빠름. 병목 현상 발생.
  - 행마다 6000회씩 검색을 하는 방법과 행마다 1000회씩 검색하는 작은 크기의 스킬 사전 파일 여러 개를 만들어 나누어 처리하는 방식 모두 테스트해보았으나, 행마다 액세스하는 횟수를 줄이는 것이 처리시간 감소에 더 효과적이었음.
